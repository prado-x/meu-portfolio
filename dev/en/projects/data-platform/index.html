PROJECT CASE STUDY

Enterprise Mass Data Generation Platform | Built at Itaú Unibanco

Tech Stack:Spring Boot 4.xJava 25JEP 491AWS ECS FargateOpenTelemetry Context PropagationDatadog Workflows

Business Problem

Coupled staging environments and extreme slowness (5-minute bottleneck) to orchestrate and generate consistent test data for Lending credit tests before running CI/CD pipelines.

Technical Solution

Replacement of blocking thread pools with native Java 25 Virtual Threads. Implementation of JEP 491 (replacing internal monitors with ReentrantLock to completely eliminate thread pinning) to trigger parallel asynchronous concurrent calls against 37 internal banking APIs. Customization of OpenTelemetry propagators to maintain Trace ID traceability across Virtual Threads in ECS Fargate.

Impact Metrics

Slashed data generation latency from 5 minutes to a surgical average of 7 seconds, maximizing CPU throughput and cutting infrastructure costs.
