PROJECT CASE STUDY

Enterprise Mass Data Generation Platform | Built at Itaú Unibanco

Tech Stack:Spring Boot 4.xJava 25JEP 491AWS ECS FargateOpenTelemetry Context PropagationDatadog Workflows

Business Problem

Ambientes de homologação acoplados e lentidão extrema (gargalo de 5 minutos) para orquestrar e gerar massas de dados consistentes para testes de crédito de Lending antes de rodar os pipelines de CI/CD.

Technical Solution

Substituição de pools de threads bloqueantes por Java 25 Virtual Threads nativas. Implementação da JEP 491 (trocando travas internas por ReentrantLock para eliminar completamente o problema de thread pinning) para disparar chamadas concorrentes assíncronas em paralelo contra 37 APIs bancárias internas. Customização de propagadores do OpenTelemetry para manter a rastreabilidade dos Trace IDs entre as Virtual Threads no ECS Fargate.

Impact Metrics

Derrubou a latência de geração de dados de 5 minutos para uma média cirúrgica de 7 segundos, maximizando o throughput de CPU e cortando custos de infraestrutura.
