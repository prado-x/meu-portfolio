iTomation Modernization: Plataforma Corporativa de Geração de Dados Massivos | Construído no Itaú Unibanco

Problema de Negócio

A plataforma legado (iTomation) era uma ferramenta em Python monolítica com execução puramente local nas máquinas dos desenvolvedores. Exigia que as squads clonassem e buildassem o código manualmente, sofria com duplicações infinitas de chamadas às APIs por falta de arquitetura e possuía uma UI rústica em Python que apenas técnicos sabiam operar, virando um enorme gargalo para homologações.

Solução Técnica

Reconstrução total do sistema utilizando arquitetura DDD (divisão em core/ e produto/), unificando microsserviços reutilizáveis. Desenvolvi o backend em Java 21 com Spring Boot 3.x e orquestrei 22 fluxos complexos (orquestrações contendo de 12 a 32 chamadas em APIs bancárias downstream). A latência caiu de 5 minutos para 7 segundos ao paralelizar todas as requisições usando Virtual Threads e Parallel Execution Services (PES) a partir do ID_Cliente retornado na primeira chamada síncrona. O frontend foi desenvolvido em Angular 18, hospedado estaticamente no S3 + CloudFront.

Métricas de Impacto

Latência média de geração de dados reduzida de 5 minutos para 7 segundos. Cobertura de testes unitários em 95% (JUnit) no backend e 92% (Jest) no frontend. Testes de mutação com 90% (Pitest) e 70% (StrykerJS). Infraestrutura automatizada via Terraform no AWS ECS Fargate e CI/CD via GitHub Actions.
