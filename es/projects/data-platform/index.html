PROJECT CASE STUDY

Plataforma de Generación de Datos Masivos Empresariales | Desarrollado en Itaú Unibanco

Tech Stack:Spring Boot 4.xJava 25JEP 491AWS ECS FargateOpenTelemetry Context PropagationDatadog Workflows

Business Problem

Entornos de desarrollo acoplados y lentitud extrema (embotellamiento de 5 minutos) para orquestar y generar datos de prueba consistentes para pruebas de crédito antes de ejecutar pipelines de CI/CD.

Technical Solution

Reemplazo de pools de hilos bloqueantes con hilos virtuales nativos de Java 25. Implementación de JEP 491 (reemplazo de monitores internos con ReentrantLock para eliminar por completo el bloqueo de hilos o pinning) para activar llamadas concurrentes asíncronas paralelas contra 37 APIs bancarias internas. Personalización de propagadores OpenTelemetry para mantener la trazabilidad de Trace ID a través de hilos virtuales en ECS Fargate.

Impact Metrics

Reducción de la latencia de generación de datos de 5 minutos a un promedio quirúrgico de 7 segundos, maximizando el rendimiento de la CPU y reduciendo los costos de infraestructura.
